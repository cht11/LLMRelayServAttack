# 创建公众号技术文章

## 工作流程

### 步骤 1：收集文章信息

向用户询问以下信息：

1. **技术内容概要**：文章要讲解什么技术或主题？
2. **标题建议**：根据内容帮用户拟定一个吸引人的标题

示例对话：
```
请告诉我这篇文章要讲解什么技术内容？

（用户回答后）

根据您的描述，小编建议以下标题：
- 「xxx 技术深度解析」
- 「从零开始理解 xxx」

您觉得哪个标题更合适，或者有其他想法？
```

### 步骤 2：选择专栏分类

根据技术内容，引导用户选择合适的专栏：

| 专栏 | 目录 | 适合内容 |
|------|------|----------|
| 逆向工程 | `reverse-engineering/` | 逆向分析、反汇编、调试技术 |
| 移动安全 | `mobile-security/` | iOS/Android 安全、App 分析 |
| 训练营 | `training-camp/` | 教程系列、学习路径、实战练习 |
| AI 与安全应用 | `ai-security/` | AI 安全、ML 在安全领域的应用 |

询问用户：
```
这篇文章属于哪个专栏？
1. 逆向工程 - 逆向分析、反汇编、调试技术
2. 移动安全 - iOS/Android 安全、App 分析
3. 训练营 - 教程系列、学习路径、实战练习
4. AI 与安全应用 - AI 安全、ML 在安全领域的应用
```

### 步骤 3：创建文章文件

确定文件名和路径：
- 文件名格式：使用小写英文和连字符，如 `hook-technology-intro.md`
- 路径：`src/<专栏目录>/<文件名>.md`

**获取当前日期**：在创建文件前，先执行系统命令获取当前日期：

```bash
date +%Y-%m-%d
```

创建文件时使用以下模板（将 `<当前日期>` 替换为上述命令输出）：

```markdown
---
创建日期: <当前日期>
作者 ID: xxx
审核 ID: xxx
是否发表: false
---

# [文章标题]

> 文章摘要：简要描述文章内容（1-2 句话）

## 引言

[在这里写开篇引言，引出要讲解的技术主题]

## 正文

[正文内容]

## 总结

[文章总结]

## 相关文章推荐

- [相关文章标题1](../专栏/文件名.md)
- [相关文章标题2](../专栏/文件名.md)
```

**元信息字段说明**：
- **创建日期**：自动获取，格式 `年-月-日`
- **作者 ID**：可询问用户，或使用默认值 `xxx`
- **审核 ID**：可询问用户，或使用默认值 `xxx`
- **是否发表**：默认 `false`，发表后改为 `true`

### 步骤 4：添加相关文章推荐

在创建文章时，需要分析 `src/SUMMARY.md` 找到 2-3 篇相关文章：

1. **读取 SUMMARY.md**：获取所有已有文章列表
2. **筛选相关文章**：根据新文章主题，选择内容相关的文章
   - 优先选择同专栏的文章
   - 也可跨专栏选择主题相关的文章
3. **生成交叉引用链接**：使用相对路径链接到相关文章

**相对路径规则**：
- 同专栏文章：`./文件名.md`
- 跨专栏文章：`../其他专栏/文件名.md`

**示例**：如果新文章在 `reverse-engineering/` 下，引用 `mobile-security/` 的文章：
```markdown
## 相关文章推荐

- [Frida Hook 入门指南](./frida-hook-intro.md)
- [iOS 应用逆向基础](../mobile-security/ios-reverse-basics.md)
```

**注意**：如果当前专栏文章较少，可以跨专栏推荐；如果整体文章很少，可以先留空或只推荐 1 篇。

### 步骤 5：更新 SUMMARY.md

在 `src/SUMMARY.md` 对应专栏下添加文章链接。

SUMMARY.md 结构：
```markdown
# Summary

## 创新规划专栏

- [逆向工程](./reverse-engineering/README.md)
  - [新文章标题](./reverse-engineering/new-article.md)  <- 添加在这里
- [移动安全](./mobile-security/README.md)
- [训练营](./training-camp/README.md)
- [AI 与安全应用](./ai-security/README.md)
```

**注意**：子文章需要缩进两个空格，添加在对应专栏 README 链接的下方。

## 完成检查清单

```
创建文章进度：
- [ ] 确定技术内容和标题
- [ ] 选择专栏分类
- [ ] 创建 md 文件（路径正确，使用模板）
- [ ] 分析 SUMMARY.md，添加 2-3 篇相关文章推荐
- [ ] 更新 SUMMARY.md（正确缩进，正确位置）
- [ ] 提醒用户参考写作规范开始撰写
```

## 完成后提示

文章创建完成后，提醒用户：

```
文章框架已创建完成！

文件位置：src/<专栏>/<文件名>.md
已添加到目录：src/SUMMARY.md

开始撰写时请注意：
- 使用"小编"作为主语
- 采用由浅入深的讲解方式
- 添加承上启下的过渡句

祝写作顺利！
```
